# ocb CLI UX Audit

Date: 2026-04-23
Status: Audit complete, pending implementation planning

Reference: clig.dev, CLI best practices (arturtamborski), git/gh/docker UX patterns

---

## Audit Scope

对比 git、gh、docker、kubectl 等 CLI 工具的使用体验，审查 ocb 全部命令的参数设计、输出格式、可发现性和 shell 集成。

---

## A. 缺失的核心能力

### A1. Shell Tab 补全（P0）

**现状**：零补全支持。用户必须记住所有命令和 session 名称。

**需要补全的场景**：

| 命令 | `<Tab>` 补全内容 |
|------|-----------------|
| `ocb <Tab>` | 所有子命令 |
| `ocb show <Tab>` | 当前目录 session 别名 |
| `ocb checkout <Tab>` | session 别名 |
| `ocb checkout -b <Tab>` | session 别名（fork 来源） |
| `ocb delete <Tab>` | session 别名 |
| `ocb unmanage <Tab>` | session 别名 |
| `ocb compact <Tab>` | session 别名 |
| `ocb model <Tab>` | 已配置模型别名 |

**实现路径**：

1. 新增 `ocb completion <shell>` 子命令
2. 内部生成 bash/zsh/fish completion script
3. session 列表动态补全需要 `ocb list --names`（新增极简输出）

**用户安装**：

```bash
ocb completion bash > ~/.local/share/bash-completion/completions/ocb
ocb completion zsh  > ~/.zfunc/_ocb
ocb completion fish > ~/.config/fish/completions/ocb.fish
```

### A2. Prompt 集成 — 显示当前 session（P0）

**现状**：用户在 shell 中无法感知当前在哪个 ocb session。

**方案（三档可选）**：

**a) 极简版**：新增 `ocb current` 命令

```bash
# 用户在 .zshrc / .bashrc 中：
PS1='$(ocb current 2>/dev/null) '$PS1
```

**b) 标准版**：输出供 starship/oh-my-zsh 消费的格式

```bash
ocb prompt
# 有 session: my-feature（exit 0）
# 无 session: 无输出（exit 1）
```

**c) oh-my-zsh plugin**：在 `contrib/` 下提供 `ocb.plugin.zsh`

```zsh
function ocb_prompt_info() {
  local name=$(ocb current 2>/dev/null)
  if [ -n "$name" ]; then
    echo "%{$fg[green]%}[ocb:$name]%{$reset_color%}"
  fi
}
RPROMPT='$(ocb_prompt_info)'
```

**d) Starship custom module**：

```toml
[custom.ocb]
command = "ocb current"
when = "which ocb"
format = "[$output]($style) "
style = "green bold"
```

## B. 参数设计问题

### B1. `-m` 标志语义不一致（P1）

| 命令 | 含义 | 语法 |
|------|------|------|
| `show -m` | 查看消息详情 | `3,5,7` 逗号分隔 |
| `compact -m` | 消息范围 | `3-8` 连字符范围 |
| `pick -m` | 选择消息 | `3,5,7` 逗号分隔 |

同一个 `-m`，三种语义。用户会在 `compact` 里写 `3,5` 然后报错。

**修复**：
- `show` → `-m` / `--messages`（保持）
- `compact` → `-r` / `--range`（改名，释放 `-m` 歧义）
- `pick` → `-m` / `--messages`（保持）
- 所有 flag 加长形式

### B2. 缺少长形式 flag（P1）

按 clig.dev 规范：所有 flag 必须有 `--long-form`。

缺失：
- `-b` → `--branch`（checkout，对齐 git）
- `-s` → `--session`（attach）

### B3. `attach` 参数语义混乱（P1）

```
ocb attach my-name -s ses_xxx     # 明确
ocb attach ses_xxx                # 隐式：正则检测 ses_ 前缀自动生成名
ocb attach                        # 隐式：附加最近 session
ocb attach --all                  # 批量
```

`name` 参数既是"期望别名"又是"session ID"，靠正则 `^ses_` 区分。

**修复**：改为纯 flag 风格

```bash
ocb attach --name my-name --session ses_xxx
ocb attach --session ses_xxx          # 自动生成名称
ocb attach --all
ocb attach                            # 附加最近的，提示输入名称
```

### B4. `rollback` 只接受 name 不接受 ref（P1）

`rollback <name>` 但其他命令都用 `<ref>`（别名或 ID）。

**修复**：改为 `rollback <ref>`。

### B5. `checkout` 无参时行为不明显（P2）

**修复**：无参数时显示当前 session 信息或列出可用 session。

## C. 输出和可发现性

### C1. 无颜色支持（P1）

所有输出纯文本 + Unicode 符号，`*` 标记当前 session 不明显。

**修复**：
- 引入 `picocolors`（零依赖 ~300B）
- 当前 session `bold green`，unmanaged `dim`，错误 `red`
- 支持 `NO_COLOR` 环境变量 + `--no-color` flag

### C2. 列表命令缺少 `--json` 输出（P0）

只有 `show` 有 `--json`。`list`、`graph`、`reflog`、`model --list` 没有。

**修复**：所有输出命令统一加 `--json` flag。对 shell completion 也很有用。

同时新增 `ocb list --names`：只输出 session 名称，每行一个，供 completion 消费。

### C3. 缺少使用示例（P2）

**修复**：在 `program.addHelpText('after', ...)` 中加 EXAMPLES 段。

### C4. 命令分类显示（P1）

当前 `ocb --help` 平铺所有命令。应按功能分组（参照 `git` 和 `gh`）：

```
Session management:
  list      List managed sessions
  show      Show session messages
  attach    Create alias for a session
  rename    Rename a session alias
  unmanage  Remove session from management
  delete    Delete a session permanently

Navigation:
  checkout  Switch active session
  open      Open session in opencode

History & branching:
  graph     Show session fork tree
  reflog    View operation history
  rollback  Rollback to a historical version

Compression:
  compact   Compress messages into LLM summary
  rebase    Interactive rebase

Cross-session:
  inject    Inject knowledge from another session
  pick      Pick specific messages

Configuration:
  model     View or set summarizer model
```

### C5. 版本号不一致（P2）

`package.json` = `0.2.1`，`src/index.ts` 硬编码 `.version("0.2.0")`。

**修复**：动态读取 `package.json` 的 `version` 字段。

## D. 命令结构

### D1. 废弃的 `origin` 命令（P2）

应加 `.hide()` 从 help 中隐藏或直接删除。

### D2. 缺少 `current` / `status` 命令（P0/P3）

```bash
ocb current           # 输出当前 session 名称（供 prompt 用）
ocb status            # 输出当前 session 信息 + 最后消息摘要
```

---

## E. 跨语言问题

### E1. 中英文混杂（P3）

CLI 界面全英文，但 compact/rebase 用 `[OCB 摘要]`，inject 注入中文文本。

**修复**：统一英文，或提供 `OCB_LANG` 环境变量。

## 实施优先级

| 优先级 | 改进项 | 工作量 | 影响 |
|--------|--------|--------|------|
| P0 | Shell completion (`ocb completion`) | 中 | 可发现性大幅提升 |
| P0 | `ocb current` + prompt 集成 | 小 | 核心体验：永远知道自己在哪 |
| P0 | `list --json` / `list --names` | 小 | completion 和脚本的基础 |
| P1 | 颜色支持 + `NO_COLOR` | 小 | 视觉可扫描性 |
| P1 | 统一 flag 长形式 | 小 | 一致性 |
| P1 | 命令分组 help 输出 | 小 | 可发现性 |
| P1 | `-m` 语义统一（compact 改 `--range`） | 小 | 避免混淆 |
| P2 | 使用示例 | 小 | 学习曲线 |
| P2 | 废弃 origin 命令 | 极小 | 清理 |
| P2 | 版本号统一 | 极小 | 正确性 |
| P3 | `ocb status` 命令 | 中 | 增量改进 |
| P3 | 中英文统一 | 中 | 国际化 |
